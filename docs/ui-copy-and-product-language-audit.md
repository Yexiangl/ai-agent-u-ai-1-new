# 全项目 UI 文案与产品语言审计

TASK-031A | 日期：2026-05-28 | 本轮只做审计，不改业务代码。

---

## 1. 总结

### 整体评价

当前 UI 文案存在三个核心问题：

1. **中英文混杂不统一**：导航同时出现 "Skill Center"（英文）和 "摸鱼中心"（中文），"Agent 对话" 混用英文前缀。
2. **技术词泄露到普通视图**：Gateway / Token / OpenClaw / HTTP / baseUrl 在非高级诊断区域可见。
3. **语气风格不统一**：首页偏正式，摸鱼中心偏段子（"合理装死""系统维护"），能力中心偏工具。

### 最需要改的模块

| 优先级 | 模块 | 问题 |
|---|---|---|
| P0 | 左侧导航 | "Skill Center" 英文孤岛，"Agent" 前缀过技术 |
| P0 | Agent 引擎页 | 普通视图暴露 Gateway/Token/OpenClaw |
| P1 | 摸鱼中心 | "装死""系统维护"太段子，老板看到不好 |
| P1 | 能力中心 | "Skill/Plugin" 英文 badge 对普通用户不友好 |
| P2 | 首页 | 基本合格，微调即可 |

## 2. 产品语言原则

### 推荐风格

| 维度 | 原则 |
|---|---|
| 专业度 | 专业但不端着，像一个靠谱的同事 |
| 轻松度 | 轻松但不低级，不用网络梗 |
| 清晰度 | 一句话说清楚，不啰嗦 |
| 用户语言 | 用户说"AI 助手"不说"Agent"，说"能力"不说"Skill" |
| 技术词 | 只在高级诊断/开发者模式出现 |
| 安全提示 | 明确但不恐吓，不甩锅 |

### 命名规则

- 导航用中文，不混英文
- 按钮用动词开头，2-4 字
- Badge 用 2-3 字短标签
- 空状态用完整句子，给出下一步

## 3. 命名统一建议

### 推荐最终方案

| 当前 | 建议 | 理由 |
|---|---|---|
| Skill Center | **能力中心** | 已有"能力中心"页面标题，导航应统一 |
| Agent 引擎 | **AI 助手** | 用户不关心"引擎"，关心"AI 助手是否可用" |
| Agent 对话 | **AI 对话** | 去掉 Agent 前缀，更自然 |
| Agent 记忆 | **记忆** | 简短，用户理解"AI 的记忆" |
| AI 文件库 | **文件库** | "AI"前缀在这里多余 |

| 首页 | **首页**（保留） | 标准 |

### 关于"摸鱼中心"命名

保留"摸鱼中心"。理由：
- 目标用户（桌面办公）对"摸鱼"有共鸣
- 改为"休息站"/"轻松一下"反而失去辨识度
- 但内部文案需要从"段子风"调整为"轻松专业风"
