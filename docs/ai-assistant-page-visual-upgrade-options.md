## 1. 当前页面问题诊断

### 当前结构

页面由 4 个并列 Card 垂直堆叠 + 2 个模态：

| 区块 | 内容 |
|---|---|
| Card 1 | AI 助手状态（连接状态 + 当前模型 + 重新检查 + 高级诊断链接） |
| Card 2 | 启用 AI 助手（条件显示，password 输入 + 一键启用） |
| Card 3 | 模型配置（密钥输入 + 档位 + 保存配置） |
| Card 4 | 本地服务诊断（服务/模型/对话状态 + 启动 + 控制台） |
| 模态 1 | Legacy 引擎应用确认 |
| 模态 2 | 高级诊断 |

### 主要问题

| # | 问题 | 严重度 |
|---|---|---|
| 1 | 4 张同权重 Card 堆叠，无主次 | 高 |
| 2 | "启用 AI 助手"和"模型配置"功能重叠，都能输入密钥 | 高 |
| 3 | 状态卡视觉不突出，和配置卡一样大 | 中 |
| 4 | 信息密度高，普通用户不知道先看哪个 | 高 |
| 5 | 诊断区和配置区视觉无区分 | 中 |
| 6 | 卡片间距均匀，缺呼吸感 | 中 |
| 7 | 缺少"高级感"——纯功能卡堆叠 | 高 |
| 8 | 已连接状态下仍显示全部配置项，冗余 | 中 |

### 现有设计系统（可复用）

- Card：`rounded-xl border bg-card shadow-sm hover:shadow-md`
- Button：`rounded-xl` + 多 variant
- Badge：6 种 tone（success/warning/danger 等）
- Switch 组件已存在
- backdrop-blur 已在 dialog 使用
- 语义色：emerald/amber/rose + dark mode

## 3. 五种视觉风格方案

### 方案 A：macOS System Settings 风格

**关键词**：系统设置感、分组列表、状态胶囊、轻分隔线

**页面结构**：
- 顶部：页面标题 + 状态胶囊（已连接/需检查）
- 分组 1「连接」：状态行 + 当前模型行
- 分组 2「模型」：密钥设置行 + 档位选择行
- 分组 3「本地服务」：服务/模型/对话三个状态行 + 操作按钮
- 底部：高级诊断入口（弱化文字链接）

**状态卡改造**：从大 Card 改为顶部一行状态胶囊 + 图标。

**模型配置改造**：改为 setting rows（左标签右控件）。

**诊断区改造**：三个状态行用分隔线分隔，右侧显示状态点。

- 适合程度：高（桌面端天然适配设置列表）
- 风险：低（结构清晰，改动可控）

---

### 方案 B：iOS Settings Card 风格

**关键词**：大圆角卡片、分组设置项、状态行、大留白、轻毛玻璃

**页面结构**：
- 分组卡片，每组 `rounded-2xl` 大圆角
- 组内是 setting rows，行间细分隔线
- 主操作（一键启用/开始对话）放底部 action area

**做成 setting rows 的内容**：
- 模型档位（右侧 segmented control）
- 各诊断状态（右侧状态点）

**底部 action area**：一键启用 / 开始对话 大按钮。

- 适合桌面端：中（iOS 风格在宽屏需控制最大宽度 ~640px 居中）
- 风险：中（大留白在桌面可能显空，需要 max-width 容器）

---

### 方案 C：Apple Dashboard 风格

**关键词**：顶部 Hero 状态面板、当前状态大卡、下方模块卡片、控制中心感

**页面结构**：
- 顶部 Hero：大卡展示"AI 助手已连接"+ 当前模型 + 主操作
- Hero 下方：3 个小指标卡（本地服务/模型连接/对话连接）横向排列
- 再下方：模型配置卡 + 诊断折叠

**Hero 展示**：大字号状态 + 渐变背景 + 状态图标。

**小指标**：本地服务/模型/对话各一个 mini 卡，显示状态点 + 标签。

- 适合程度：高（视觉冲击力强，最有"高级感"）
- 风险：中（Hero 在未配置状态需要不同设计，状态切换要处理好）

---

### 方案 D：Pro Console Lite 风格

**关键词**：专业但不吓人、左状态摘要、右配置区、高级诊断折叠

**页面结构**：
- 两栏布局：左侧窄栏状态摘要（sticky），右侧宽栏配置/诊断
- 左栏：连接状态 + 当前模型 + 主操作
- 右栏：模型配置 + 诊断

**兼顾普通用户和高级诊断**：左栏极简给普通用户，右栏诊断可折叠给运营。

**避免技术感过重**：左栏不显示技术词，技术细节在右栏折叠区。

- 适合内测还是正式版：正式版（两栏在小屏需要响应式降级）
- 风险：中高（响应式复杂度，移动端需堆叠）

---

### 方案 E：Minimal Setup First 风格

**关键词**：极简主路径、只突出已连接/一键启用/开始对话、配置诊断全弱化

**页面结构**：
- 已连接状态：一个居中大状态 + "开始对话"主按钮 + "管理配置"弱链接（点击展开配置/诊断）
- 未配置状态：一个居中卡片 + 密钥输入 + 一键启用
- 模型配置：默认折叠，点"管理配置"展开
- 诊断：默认隐藏，仅异常时自动展开

**已连接展示**：大绿勾 + "AI 助手已连接" + 开始对话。

**未配置展示**：聚焦密钥输入 + 一键启用。

- 风险：中（配置/诊断折叠后，运营排查需要多一步；但最符合无技术客户）

## 5. 推荐方案

### 最推荐：方案 A（macOS System Settings）+ 方案 C 的 Hero 顶部

**组合理由**：

1. **方案 A 的分组列表**最适合"配置页"语义，桌面端天然适配，实现成本可控，风险低。
2. **吸收方案 C 的 Hero 顶部**——用一个状态 Hero 突出"已连接/当前模型/主操作"，解决当前"状态卡不突出"的问题，提供高级感。
3. **吸收方案 E 的状态自适应**——已连接时弱化配置项，未配置时聚焦密钥输入。

**为什么不选纯 C**：Hero + 多卡片在不同状态（已连接/未配置/服务未运行）下需要维护多套 Hero 设计，成本和风险更高。

**为什么不选 D**：两栏响应式复杂度高，不适合当前阶段。

**为什么不选纯 E**：折叠诊断对客服/运营排查不友好。

### 推荐优先级

1. 方案 A + C-Hero（推荐）
2. 方案 C
3. 方案 E
4. 方案 B
5. 方案 D

### 是否先只改 AI 助手页

**是。** 建议：
1. 先在 AI 助手页落地方案 A+C-Hero
2. 验证视觉规范和用户反馈
3. 提炼可复用规范（见第 7 节）
4. 再推广到首页/能力中心/用量等页面

## 6. 推荐信息架构

### 顶部区域（Hero）

```
┌─────────────────────────────────────┐
│  ●  AI 助手已连接                     │
│     当前模型：质量优先                 │
│                    [开始对话] [重新检查]│
└─────────────────────────────────────┘
```

- 大状态图标（绿勾/橙色感叹号）
- 状态文案（已连接/需要配置/服务未运行）
- 当前模型
- 主操作：已连接→开始对话；未连接→引导配置

### 中间区域（设置分组）

「模型」分组：
- 模型访问密钥（password 行 + 保存）
- 模型档位（segmented：速度 / 质量）

已连接状态下此组可折叠为摘要行。

### 诊断区域（设置分组）

「本地服务」分组：
- 本地服务状态（状态点 + 标签）
- 模型连接状态（状态点 + 标签）
- 对话连接状态（状态点 + 标签）
- 操作：启动本地服务 / 打开控制台

### 高级区域（折叠）

- 高级诊断（模态保持）
- 内部状态 / 技术细节
- 控制台说明
- Legacy 引擎

## 7. 视觉规范建议

| 维度 | 规范 |
|---|---|
| 卡片圆角 | 分组卡 `rounded-2xl`，内部行无圆角 |
| 阴影 | 默认 `shadow-sm`，hover `shadow-md`，Hero 可 `shadow-lg` |
| 边框 | `border border-border/60`，分组内行用 `divide-y` |
| 背景层级 | 页面 `bg-background`，卡 `bg-card`，Hero `bg-gradient-to-br from-primary/5` |
| hover | 可点行 `hover:bg-muted/50` |
| active | `active:scale-[0.98]` 仅主按钮 |
| disabled | `opacity-50 cursor-not-allowed`（已有） |
| 状态 badge | 复用现有 Badge tone |
| 成功色 | emerald-500 |
| 警告色 | amber-500 |
| 错误色 | rose-500 |
| 毛玻璃 | 仅 Hero 顶部和模态用 `backdrop-blur`，不滥用 |
| spacing | 分组间 `space-y-4`，行内 `py-3 px-4` |
| 字号层级 | Hero 标题 `text-lg font-semibold`，行标签 `text-sm`，辅助 `text-xs text-muted-foreground` |
| 图标风格 | 保持 lucide-react，状态点用 `h-2 w-2 rounded-full` |

### 设置行（Setting Row）模板

```
左：图标 + 标签 + 副标题
右：控件（开关/segmented/状态点/按钮）
分隔：divide-y divide-border/60
高度：min-h-[52px] py-3 px-4
hover：hover:bg-muted/40（仅可点行）
```
